### Import alias ‚Äî important

Imports use **`#/`** (Node subpath imports), NOT `@/`. e.g. `import { Button } from '#/components/ui/button'`,
`import { cn } from '#/lib/utils'`. Maps to `./src/*` (see `package.json` `imports` + `components.json` aliases).

### Project layout ‚Äî feature-based

Code is organised by **feature**, not by file type. Each feature owns its components, hooks, and
logic; a shared kernel holds cross-cutting pieces.

```
src/
‚îú‚îÄ‚îÄ features/
‚îÇ   ‚îú‚îÄ‚îÄ data/         # the data domain ‚Äî types.ts, schemas.ts, defaults.ts, storage.ts,
‚îÇ   ‚îÇ                 #   PortfolioDataContext.tsx, usePortfolioData.ts (imported by both other features)
‚îÇ   ‚îú‚îÄ‚îÄ portfolio/    # public site ‚Äî components/ (sections + Navbar/Footer/ScrollProgress + Project/SkillCard)
‚îÇ   ‚îÇ                 #   and hooks/ (useTypingEffect, useActiveSection, useScrollProgress)
‚îÇ   ‚îî‚îÄ‚îÄ admin/        # dashboard ‚Äî components/ (AdminLayout, AdminAuthGate, form modals, dnd list, inputs)
‚îú‚îÄ‚îÄ components/
‚îÇ   ‚îú‚îÄ‚îÄ ui/           # shadcn primitives (keep here ‚Äî components.json points at it)
‚îÇ   ‚îú‚îÄ‚îÄ shared/       # feature-agnostic presentational: Reveal, SectionHeader, CountUp
‚îÇ   ‚îî‚îÄ‚îÄ layout/       # app-shell: ThemeProvider, ThemeToggle
‚îú‚îÄ‚îÄ hooks/            # cross-cutting: useScrollAnimation
‚îú‚îÄ‚îÄ lib/              # animations/, utils.ts (cn), utils/constants.ts
‚îî‚îÄ‚îÄ routes/           # thin TanStack route files that compose feature components
```

Conventions: intra-feature and cross-feature imports use **specific file paths** (e.g.
`#/features/data/types`, `#/features/data/usePortfolioData`) ‚Äî there are no barrel `index.ts` files
(avoids circular deps and `import/no-duplicates` churn). Files inside `features/data/` import each
other via `#/features/data/<file>` too. Routes are the only place that "compose" multiple features.

### pnpm gotcha (already fixed, don't undo)

`pnpm-workspace.yaml` sets `verifyDepsBeforeRun: false` and lists `unrs-resolver` under
`onlyBuiltDependencies`. Without this, pnpm's ignored-build guard makes every `pnpm <script>` exit
non-zero (it also silently aborts `shadcn add`). If you hit `ERR_PNPM_IGNORED_BUILDS`, remove
`node_modules/.modules.yaml` and re-run `pnpm install` rather than reverting the config.

## Architecture ‚Äî the parts that matter

### Data flow is the core design (read this first)

All editable content lives in **one** `PortfolioData` object persisted to `localStorage` under key
`portfolio_data`. There is no server. The flow is strictly one-directional:

```
defaults.ts (seed) ‚îÄ‚îÄ‚ñ∫ storage.ts (load/save/import/export) ‚îÄ‚îÄ‚ñ∫ PortfolioDataContext (Context + reducer)
                                                                          ‚îÇ
                            usePortfolioData() ‚óÑ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îò
                                   ‚îÇ
                  every section + every admin page reads here
```

Hard rules that the whole codebase depends on:

- **Components NEVER import `defaults.ts` directly.** They read content only through
  `usePortfolioData()`. `defaults.ts` is the first-run seed and the reset target ‚Äî nothing else.
- **Every mutation goes through the reducer** via a named `DataAction` command
  (`UPDATE_PERSONAL_INFO`, `ADD_PROJECT`, `DELETE_PROJECT`, `REORDER_PROJECTS`, `IMPORT_DATA`,
  `RESET_TO_DEFAULTS`, ‚Ä¶). No component mutates state directly.
- **Each dispatch auto-saves to localStorage** (and bumps `lastUpdated`). Saving is a side effect of
  the reducer/provider, not the caller's job.
- **Import JSON must be Zod-validated before it is applied.** Export is a plain JSON download.

Single source of truth for shapes: TypeScript interfaces in `src/features/data/types.ts` and Zod
schemas in `src/features/data/schemas.ts` must stay aligned ‚Äî `schemas.ts` enforces this at compile
time via the `SchemaTypeChecks` `Equal<>` tuple (a drift becomes a `tsc` error). `PortfolioData` =
`{ personalInfo, skills, projects, experiences, education, lastUpdated }`. (`src/lib/utils.ts` with
`cn()` exists from shadcn.)

### Admin access

`/admin/*` is gated by a password dialog comparing against `import.meta.env.VITE_ADMIN_PASSWORD`
(set in `.env.local`, never committed). A valid session is stored in `sessionStorage['admin_auth']`
(session, not local). Exclude `/admin` from sitemap/SEO.

### i18n (Paraglide) ‚Äî read before touching UI copy

The public site is bilingual **FR/EN** via **Paraglide JS** (`paraglideVitePlugin` in `vite.config.ts`).
This is the officially-tested TanStack Start integration; do **not** swap it for i18next.

- **Base locale `fr` is unprefixed** (`/`, `/projects`), **`en` is prefixed** (`/en/‚Ä¶`). One physical
  route serves both ‚Äî the router `rewrite` (`src/router.tsx`) de-localizes the URL for matching and
  re-localizes generated hrefs, so internal `Link`/`navigate` stay in the active locale automatically.
- **Detection order:** `url ‚Üí cookie ‚Üí preferredLanguage (Accept-Language) ‚Üí baseLocale`. SSR runs
  through `paraglideMiddleware` in `src/server.ts` (pass the **original** `req` ‚Äî the router does the
  rewrite; passing the modified request double-delocalizes and loops).
- **Messages are typed, compiled functions:** `import { m } from '#/paraglide/messages'` ‚Üí `m.nav_about()`,
  params as `m.education_item({ number })`. **Never** type a translation key as a string.
- **Source messages:** `src/features/i18n/messages/{fr,en}/<namespace>.json` (namespaces:
  `common, nav, hero, projects, experience, education, skills, contact, meta`). The runtime `m`
  namespace is **flat**, so keys are globally unique ‚Äî prefix by namespace (`nav_`, `hero_`, ‚Ä¶).
  `src/paraglide/` is **generated** (gitignored, eslint-ignored) ‚Äî never edit; the Vite plugin
  recompiles on dev/build, or run `pnpm exec paraglide-js compile --project ./project.inlang
  --outdir ./src/paraglide --strategy url cookie preferredLanguage baseLocale --emit-ts-declarations`.
- **Module-scope `m.*()` is an SSR footgun:** it resolves once at import and leaks that locale across
  requests. Call messages **at render** (or build locale-dependent objects like Zod schemas inside the
  component). For arrays of labels, store the **getter** (`label: () => m.x`) and call it in render ‚Äî
  see `NAV_ITEMS` and `LEVEL_LABEL`.
- **Translate static UI copy only.** All `usePortfolioData()` content (personalInfo, projects,
  experiences, skills, education) and `/admin` stay French-only ‚Äî do **not** add locale variants to
  `types.ts`/`schemas.ts`/the reducer. Data-enum values shown directly (project `type` filters, skill
  categories) stay untranslated to match their (untranslated) data; only UI-only labels are localized.
- **Switcher:** `LanguageToggle` (`src/components/layout/`) calls `setLocale()` (cookie + full nav).
- **SEO:** localized `<title>`/description + `hreflang` (lowercase key ‚Äî TanStack renders link attrs
  verbatim) + canonical live in the `index` route head; `/admin` head sets `noindex`. `<html lang>` is
  `getLocale()` in `__root.tsx`. Locale logic is unit-tested in `src/features/i18n/localeRedirect.test.ts`.

### Layout / routing

File-based routes live in `src/routes/`; `src/routeTree.gen.ts` is **generated** (don't edit).
`src/routes/__root.tsx` is the root layout ‚Äî wrap the app there in `PortfolioDataProvider` +
ThemeProvider (the TanStack Query provider is already wired via `src/integrations/tanstack-query/`).
Global CSS is `src/styles.css` (Tailwind v4, `@import`-based ‚Äî no `tailwind.config.ts`).

Target routes: portfolio (`index`, `projects`, `experience`, `contact`) render section components;
`admin/*` render CRUD pages inside `AdminLayout` (separate dark admin theme: bg `#0F172A`, sidebar
`#1E293B`).

## Conventions

- **SOLID / Container-Presenter**: presentational components (`ProjectCard`, `SkillBadge`) only
  render; orchestration and data access live in containers/hooks. Extend via props
  (`variant`/`size`/`as`), don't fork components.
- Reusable logic ‚Üí custom hooks. Cross-cutting hooks live in `src/hooks/` (`useScrollAnimation`);
  feature hooks live with their feature (`src/features/portfolio/hooks/`, `src/features/data/usePortfolioData.ts`).
- No magic numbers ‚Äî named constants in `src/lib/utils/constants.ts`. Comments explain *why* only.
- Animation: shared variants (`fadeUpVariants`, `staggerContainerVariants`) and `useScrollAnimation`
  (`useInView`, `once: true`). **Respect `prefers-reduced-motion` everywhere.**
- Theme: dark by default, class-strategy toggle. Accents blue `#3B82F6` / teal `#14B8A6`.

## Quality bar (from spec)

Zero `any`; types ‚áÑ Zod schemas aligned; all mutations via reducer with auto-save; import validated;
reset requires confirmation; lazy images; WCAG AA contrast + full keyboard nav + `aria-label` on
icon-only controls; per-page meta tags and a JSON-LD Person schema generated from `personalInfo`.

**React 19**: do NOT hand-write `useMemo`/`useCallback`/`React.memo` ‚Äî the React Compiler handles
memoization. Write plain components; only reach for manual memoization if profiling proves a need.
